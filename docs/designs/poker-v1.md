## Glossary

- **Chips:** In-game betting currency used during poker hands. Each player starts each hand with a fixed stack of 1,000 chips. Chips do NOT persist across hands — every new hand resets to 1,000. The 3D chip stack visualization tracks changes within a single hand only.
- **Competition Points (CP):** Earned 100 per hand completed. Used for weekly leaderboard ranking. Tiebreaker: highest single-hand chip win. Reset to 0 every Monday 00:00 UTC via server-side scheduled check (on hand completion and on server startup, compares current week > stored week). Minimum 10 hands to qualify for weekly prize. Top 1 player wins.
- **[Sync]:** s&box networked property — continuously replicated from server to all clients. Use for game state (chip counts, cards, pot).
- **[Broadcast]:** s&box one-shot RPC event sent from server to all clients. Use for ephemeral events (animations, sounds, emotes).

## Table Configuration

- **Players per table:** Min 2, max 6. Hand starts automatically when 2+ seated players are ready.
- **Blinds:** Fixed — Small Blind = 10 chips, Big Blind = 20 chips. No blind escalation at MVP.
- **Starting stack:** 1,000 chips per hand (reset every hand).
- **Dealer button:** Rotates clockwise each hand.
- **Last player standing:** If all opponents disconnect/fold, remaining player wins the pot immediately and earns CP for the hand.

## Prize & Payout System

### MVP Payout Flow (Manual)
1. Weekly leaderboard locks Monday 00:00 UTC
2. Winner is displayed on lobby screen and announced in Discord
3. Winner DMs developer via Discord with PayPal email or gift card preference
4. Developer sends prize within 48 hours via digital gift card (Amazon/Steam — PRIMARY) or PayPal Friends & Family (last resort)
5. Prize delivery is confirmed in Discord (public accountability)

### Payout Risks & Mitigations
- **PayPal ToS:** PayPal restricts gambling-related payouts. This is NOT gambling (no entry fee, no player wager) — it's a developer-sponsored prize. However, PayPal may still flag it. **Fallback:** Digital gift cards (Amazon, Steam) as primary; PayPal as optional.
- **Fraud:** Winner verification via s&box player ID + Discord account. No anonymous winners.
- **Disputes:** Developer has final say on prize disputes. Anti-collusion audit logs provide evidence.

## Legal & Compliance

**Pre-launch requirements (blocking):**
1. Confirm s&box EULA permits developer-sponsored prize competitions (no player entry fee = not gambling)

**Post-launch (non-blocking):**

## Disconnect Handling

- **During betting round (player's turn):** Action timer (30s) continues. On timeout: auto-check if possible, auto-fold if there's a bet to call.
- **During betting round (not player's turn):** No immediate action. If player reconnects before their turn, they continue normally.
- **After disconnect, hand continues:** Disconnected player is treated as all-in for the chips they have already committed to the pot at the moment of disconnect. A side pot is created for any further betting among active players. No additional chips deducted from the disconnected player.
- **Reconnection:** Player rejoins same seat if reconnected within 60 seconds. After 60s, seat is freed; player can sit at a new seat for the next hand.
- **Competition Points:** CP is awarded only for hands where the player was present at showdown OR was disconnected after committing chips (all-in treatment). Intentional disconnect-to-dodge does not earn CP.

- State machine emits serialized snapshot at each transition (enables hand replay)
- All hand evaluation and bet resolution server-side only (validated in Week 1 spike)

## Reviewer Concerns — RESOLVED

Issues found during 3 rounds of adversarial spec review (4/10 → 6/10 → 6/10). All resolved during engineering review (2026-03-23):

1. **Split pot CP:** RESOLVED — Each player who wins any portion of a split pot earns the full 100 CP (not divided). Example: 3-way split = 300 CP total awarded.
2. **Spectator data exposure:** RESOLVED — Hole cards are delivered via per-player owner-only RPC, never stored in [Sync] properties. Spectators physically cannot access hole card data. Community cards and chip counts use [Sync] (public info).
3. **CP tiebreaker depth:** RESOLVED — Tiebreaker chain: (1) Total CP → (2) Highest single-hand chip win → (3) Earliest best-hand timestamp. If all three are identical, prize is split equally.
4. **Disconnect CP award:** RESOLVED — Disconnected all-in player earns CP ONLY if they win the hand or a side pot. A disconnected player who loses gets no CP for that hand.
5. **Scope table inconsistency:** RESOLVED — Seasonal League row in Scope Decisions table changed to "DEFERRED (Phase 2)".
6. **Payout dry-run timing:** RESOLVED — Payout dry-run is Week 9 only (per Pre-Launch Gate). Removed the Week 8 reference.
7. **State machine snapshots:** RESOLVED — Snapshot emission is built into the state machine from Week 2 (when state machine is first constructed). Each state transition serializes game state. Week 6 just adds the replay UI that reads snapshots.
8. **CP reset edge case:** RESOLVED — CP reset uses ISO week-number comparison (current_week > stored_last_reset_week). Triggers on: every hand completion, server startup, and leaderboard request. Idempotent — can run 1000 times safely.
9. **Seasonal schema foundations:** RESOLVED — CP storage schema is season-aware from day one: (player_id, season_id, week_number, points). Season 1 starts at launch.
10. **PayPal F&F risk:** RESOLVED — Digital gift cards (Amazon, Steam) are PRIMARY payout method. PayPal F&F is last resort only.
11. **Spike failure buffer:** RESOLVED — Acknowledged as escalation-worthy risk. If Week 1 spike fails (+2 weeks), the 2-3 week buffer is consumed. Mitigation: spike is the FIRST task in Week 1. If it fails by end of Week 1, the team has a full week to evaluate the fallback (server-authoritative polling model) before committing buffer time.
12. **All-fold transition:** RESOLVED — State machine includes explicit "all fold" transition from any betting round directly to CP Award, bypassing Showdown. Triggered when all players except one have folded.
13. **Hand replay cap:** RESOLVED — Initial cap is 50 hands (configurable via constant, not hard-coded magic number). Validated during Week 9 playtesting. Can be adjusted without code change.
14. **Timer action wording:** RESOLVED — Disconnect handling section now references Key Implementation Decisions #2: "30 seconds + auto-check if possible, auto-fold if there's a bet to call."

## Engineering Review Decisions (2026-03-23)

Decisions made during /plan-eng-review, integrated into implementation approach:

### Architecture Decisions
1. **Hole card security:** Per-player owner-only RPC. Hole cards NEVER exist in [Sync] properties. Community cards use [Sync].
2. **State machine snapshots:** Built into state machine from Week 2 (not retrofitted in Week 6). Each transition emits serialized snapshot.
3. **Weekly CP reset:** ISO week-number comparison. Idempotent check on hand completion, server startup, and leaderboard request.
4. **Audit log storage:** JSON lines per daily file on server disk. Supports grep-based "player X folded to player Y" queries.
5. **Table management:** Fixed tables pre-placed in lobby scene (3-4 tables for MVP). No dynamic spawning. Players walk up and press E to sit.
6. **Game state ownership:** Each PokerTable Component owns its own state machine. GameManager stays lightweight — only tracks lobby vs. seated.
7. **Crash recovery:** Accept hand loss on server crash. CP is persisted to disk (s&box leaderboard API + local fallback) after EVERY hand award. Max data loss: 100 CP (one hand).
8. **CP schema:** Season-aware from day one: (player_id, season_id, week_number, points).
9. **Action timer:** Server-authoritative deadline timestamp via [Sync]. Client interpolates countdown display locally.

### Failure Handling
10. **Hole card RPC failure:** If RPC delivery isn't acknowledged within 2s, re-send. If still fails, force-fold the player for that hand.
11. **Audit log write failure:** If file write fails, log warning + buffer in memory. If buffer exceeds 1000 events, halt new hands until resolved.

### Test Infrastructure
12. **Test project:** xUnit test project (supercasino.tests.csproj) created in Week 1 alongside hand evaluator.
13. **Coverage requirement:** 100% unit test coverage for ALL server-side game logic: hand evaluator, side pot calculator, betting round state machine, CP system, and disconnect handling.
